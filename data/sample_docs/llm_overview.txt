Large Language Models: An Overview

What is a Large Language Model?
A Large Language Model (LLM) is a type of artificial intelligence model trained on vast amounts of
text data. LLMs learn statistical patterns in language, enabling them to generate, summarise,
translate, and reason about text. Well-known examples include GPT-4, Claude, Gemini, and LLaMA.

How are LLMs trained?
Training happens in two main phases:

1. Pre-training: The model is exposed to a large corpus of text (books, websites, code) and learns
   to predict the next token. This phase requires enormous compute — often thousands of GPUs running
   for weeks. The result is a base model that understands language but has no particular behaviour.

2. Fine-tuning / RLHF: The base model is refined using human feedback (Reinforcement Learning from
   Human Feedback). Human raters score model outputs, and the model is updated to produce more
   helpful, accurate, and safe responses.

Key concepts

Tokens: LLMs do not process raw characters. Text is first broken into tokens — subword units that
balance vocabulary size and sequence length. "Unbelievable" might become ["Un", "believ", "able"].

Context window: The maximum number of tokens an LLM can process at once. Early GPT-3 had a 4,096
token window. Modern models support 128k or more, enabling analysis of entire codebases or books.

Temperature: A sampling parameter that controls randomness. Temperature=0 makes the model
deterministic (always picks the most likely token). Higher values (0.7–1.0) produce more creative
but less predictable output.

Embeddings: A numerical representation of text as a high-dimensional vector. Similar texts have
vectors that are geometrically close. Embeddings power semantic search, clustering, and RAG systems.

Retrieval-Augmented Generation (RAG)
RAG improves LLM accuracy by retrieving relevant documents at inference time and including them in
the prompt. This grounds the model's answer in real source material and reduces hallucination. A
typical RAG pipeline: chunk documents → embed chunks → store in a vector database → at query time,
embed the question → retrieve top-k similar chunks → pass them as context to the LLM.

Limitations of LLMs
- Knowledge cutoff: LLMs have a training cutoff date and do not know about events after it.
- Hallucination: Models can generate plausible-sounding but incorrect information.
- Context length: Very long documents must be chunked or summarised, which can lose detail.
- Cost: Inference at scale is expensive; prompt caching and batching help significantly.
- Bias: Models reflect biases present in their training data.
